Fast, spatial PDF parsing in Rust.

Extract text with columns, tables, and layout intact — plus Tesseract OCR for scans and format conversion for Office docs. Ships as a single self-contained binary (PDFium embedded).

Why spdf

Most PDF-to-text tools collapse whitespace, shuffle columns, and emit one giant line-salad. spdf keeps the geometry.

Column-aware projection

Tables, two-column layouts, sidebars, and indented blocks come back in reading order — not a flat string of tokens.

Faux-bold & shadow dedup

PDFs that "draw text twice" to fake bold used to produce TaTax Infofo. spdf outputs Tax Info.

Word reconstruction

Per-glyph extraction is stitched back into words (1 8 3 6 → 1836) with a liteparse-compatible merge heuristic.

QR / microprint filter

The hundreds of tiny numeric glyphs that encode a QR code are auto-dropped so they don't destroy the surrounding table.

Tesseract OCR, built in

Per-word bboxes, thread-local engine cache for speed, parallel pages via rayon. Or plug any HTTP OCR server.

One static binary

PDFium is embedded at build time (~7.5 MB) and extracted to a user cache on first run. No runtime setup.

Spatial precision

A spatial parser's job is to put each word in the right place on the page, not just to emit the right tokens. We also compare every matched word's bounding box against the raw-tesseract ground truth and report mean IoU, the fraction of matches that clear the COCO-style IoU ≥ 0.5 "well localised" bar, and the mean centroid error in PDF points.

Install

From source

git clone https://github.com/Fanaperana/spdf
cd spdf
make install       # release build + copy to ~/.local/bin

With OCR

# Debian / Ubuntu
sudo apt-get install -y libtesseract-dev libleptonica-dev \
                        clang tesseract-ocr tesseract-ocr-eng
make install-ocr

Quick start

# Plain text with preserved layout
spdf parse invoice.pdf --no-ocr --format text

# Structured JSON with per-glyph bounding boxes
spdf parse invoice.pdf --format json > out.json

# OCR a scanned PDF (uses embedded Tesseract)
spdf parse scan.pdf --ocr-language eng

# Use an external OCR server instead
spdf parse scan.pdf --ocr-server-url http://localhost:8000

# Batch convert a directory
spdf batch-parse ./inputs ./outputs --format text
